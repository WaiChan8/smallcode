# Iterative Retrieval Ladder

Before reading files, climb this ladder — stop when you have enough context:

1. `memory_load` — past decisions, gotchas, workflows
2. `search` / `graph_search` — find paths and snippets
3. `read_file` — one targeted file
4. Full repo reads — last resort only

Small models lose quality when they dump whole files into context. Memory and search are cheaper than reads.
